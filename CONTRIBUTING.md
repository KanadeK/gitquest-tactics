# Contributing

Thank you for improving GitQuest Tactics. Discuss substantial level or engine changes in an issue first. Keep the graph engine deterministic, add a focused regression test for every bug, and run `npm run lint && npm run typecheck && npm run test:coverage && npm run test:e2e && npm run build` before opening a pull request.

Do not add credentials, user repositories, telemetry, copied lesson material, or a command path that claims success without changing and validating graph state. New levels need a starter graph, allowlist, objective, and automatic solution where feasible.
